# Changelog

모든 주요 변경 사항을 이 파일에 기록합니다.

## v0.1.25
- 메뉴바 메뉴를 그룹(**번역 · 도구 · 설정 · 앱**)으로 재정렬하고 네이티브 섹션 헤더 적용 — 빈 이중 구분선 버그 수정
- 서브메뉴(입력 장치 · 번역 모델)에 현재 선택값을 부제로 표시(macOS 14.4+), `번역 시작`/`번역 종료`·`번역 입력창` 항목에 단축키 힌트 표시
- 환경설정 용어집 입력칸에 라운드 외곽선 + 예시 placeholder + 인식된 용어 개수/형식 오류 표시
- 저장 폴더 경로 박스에 외곽선 추가, 표시 옵션 슬라이더 라벨 폭 정렬

## v0.1.24
- 모델 리스트를 한국어 특화 경량 구성으로 교체 — **Gemma 3 4B(기본) · EXAONE 4.0 1.2B · HyperCLOVAX SEED 1.5B · Kanana 1.5 2.1B**
- EXAONE 4.0 1.2B(LG·최신·812MB) / HyperCLOVAX SEED 1.5B(Naver) / Kanana 1.5 2.1B(Kakao) 추가 — 모두 Ollama에서 pull 가능함을 확인
- 기존 Gemma 3 12B / Qwen 2.5 7B / EXAONE 3.5 7.8B 선택지 제거 — 저장된 설정값은 Gemma 3 4B로 자동 폴백

## v0.1.23
- 모델 리스트를 **Gemma 3 (4B / 12B) · Qwen 2.5 7B · EXAONE 3.5 7.8B**로 복원, 기본값 **Gemma 3 4B**
- EXAONE 3.5 2.4B 기본값(v0.1.21)은 출력이 산만해 되돌림 — 저장된 설정값은 Gemma 3 4B로 자동 폴백
- Gemma 4(gemma4:*) 선택지 제거

## v0.1.22
- Ollama 서버 오류(모델 로드 실패 등)를 조용히 삼키지 않고 노치에 `〔번역 오류〕 …`로 표시 — 기존엔 응답의 error 필드 디코딩 실패로 번역이 “듣는 중…”에서 멈춰 있었음
- 참고: EXAONE 3.5는 head 차원이 80이라 Ollama `OLLAMA_KV_CACHE_TYPE=q8_0`와 비호환 — KV 캐시를 `f16`으로 두어야 함(Gemma/Qwen은 q8_0 정상)

## v0.1.21
- 기본 번역 모델을 **EXAONE 3.5 · 2.4B**(LG AI · 한↔영 이중언어 특화)로 교체 — Gemma 4 E4B 대비 빠르고 한국어 번역 품질 향상
- 모델 선택지에 EXAONE 3.5 2.4B / 7.8B 추가(Ollama 태그 `exaone3.5:2.4b` / `exaone3.5:7.8b`)
- 기존 Gemma 4(E2B/E4B/12B) 선택지는 유지 — 저장된 설정값은 그대로 보존

## v0.1.20
- 번역 모델을 **Gemma 4** 계열로 전면 교체 — E2B / E4B / 12B 중 선택(기본 **E4B**)
- 기존 Gemma 3 · Qwen 2.5 · EXAONE 3.5 선택지 제거
- Ollama 태그: `gemma4:e2b` / `gemma4:e4b` / `gemma4:12b`
- 기존 설정값은 자동으로 E4B로 폴백됨(새 모델 pull 필요)

## v0.1.19
