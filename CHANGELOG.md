# Changelog

모든 주요 변경 사항을 이 파일에 기록합니다.

## v0.1.20
- 번역 모델을 **Gemma 4** 계열로 전면 교체 — E2B / E4B / 12B 중 선택(기본 **E4B**)
- 기존 Gemma 3 · Qwen 2.5 · EXAONE 3.5 선택지 제거
- Ollama 태그: `gemma4:e2b` / `gemma4:e4b` / `gemma4:12b`
- 기존 설정값은 자동으로 E4B로 폴백됨(새 모델 pull 필요)

## v0.1.19
- 번역 지연 개선: 발화마다 나가던 Ollama 헬스체크(/api/version) 왕복을 3초 캐시로 제거
- 스트리밍 정리(cleanup) 최적화 — 접두어가 있을 때만 치환해 부분 갱신 시 불필요한 문자열 할당 감소
- 번역 재시도 시 노치 오버레이가 처음부터 다시 채워지며 깜빡이던 현상 제거(최종 결과만 반영)
- 온디바이스 음성 인식 실패 시 서버 인식으로 1회 자동 폴백(무한 재시작 방지)

## v0.1.18
- 환경설정 ‘번역 모델’에 Ollama 서버 제어 추가 — 서버 상태 표시 + 시작/중지/재시작/다시 점검 버튼(brew services)
- 이제 터미널 없이 앱에서 직접 Ollama 서버를 켜고 끕 수 있음

## v0.1.17
- v0.1.16(MLX 인프로세스 Gemma) 롤백 — MDM/EDR 관리 환경에서 실행 차단 이슈로 Ollama 기반 v0.1.15 번역 경로로 복귀
- Package.swift의 MLX 의존성(mlx-swift-lm / swift-huggingface / swift-transformers) 재비활성화, GemmaTranslator는 Ollama 경로 사용
- 배포용 zip 생성(build-app.sh --zip) 등 v0.1.16 이후 빌드 도구 개선은 유지

## v0.1.16 (철회됨 / withdrawn)
- MLX(인프로세스 Gemma) 로컬 번역 경로 활성화 시도 — Ollama 없이 앱 프로세스 안에서 직접 번역
- MLXLLM/MLXLMCommon 저장소 이전 대응: mlx-swift-examples → mlx-swift-lm
- 참고: ad-hoc 서명 + 인프로세스 모델 다운로드·Metal JIT 실행으로 MDM/EDR 차단 발생 → v0.1.17에서 롤백
